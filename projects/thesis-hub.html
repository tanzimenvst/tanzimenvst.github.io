Why I Built This

While visualizing my own thesis data, I realized that every student and researcher could benefit from a shared platform that lets them spatially visualize academic research. This idea inspired me to develop Thesis Hub — a centralized academic web app where anyone can explore, visualize, and upload research work interactively on a map.

Universities and departments can use it to map their alumni theses, students can identify unexplored locations, and researchers can easily find spatially related past work. It can turn institutional research data into an interactive, discoverable map-based experience.

Full-Stack Problem Solving: This project demonstrates my ability to design, develop, and deploy a complete data-driven web application — from backend architecture to interactive frontend visualization, with integrated authentication, content management, and email automation.
How It Works
  • All thesis projects are displayed on an interactive map using markers.
  • Users can filter by university, department, degree, and roll number.
  • Each marker displays thesis title, abstract, study area, sampling points, and other spatial results.
  • Users can upload their own theses and visualize result maps.
  • User authentication system with registration, login, and email-based verification.
  • Password reset via time-limited secure link sent directly to user email.
  • Admin panel (CMS) to manage submitted theses, verify uploads, and maintain datasets.
  • Email notifications automatically sent to admins for any updates.

Hosting & Performance

This project is hosted on Render’s free-tier Singapore server to make it publicly accessible as a non-commercial showcase of my visualization.

Render’s free-tier services automatically go to sleep if inactive for 15 minutes — causing a cold start delay of 1–2 minutes when re-accessed. To keep the app responsive and available 24/7, I implemented a background “keep-alive” mechanism that periodically pings the server, effectively preventing it from sleeping. This ensures that users can access the live app instantly at any time.

Since the app is hosted on free-tier services, it may be slightly slow or temporarily offline at times. It currently performs best on desktop — mobile view is not yet fully supported.
