Why I Built This

I developed this app to have the exact functionalities that match the way I manage my personal finances — including grouped categories, monthly summaries, editable allocations, and offline access — all designed to work exactly the way I want.

And the cool thing is — since I built it from scratch, I can add or modify any functionality anytime I need — with full control over every part of it.

Practical Problem Solving: This project reflects my capability to turn a personal need into a functional and maintainable solution, built from the ground up with performance, usability, and privacy in mind.
How It Works
  • The app provides a straightforward interface for tracking daily expenses where I can input spending by category, amount, and date.
  • The main view dynamically updates to show monthly summary where I also can switch between months and view previous monthly expense summaries.
  • Real-time color-coded remaining balances (🟩 = Under budget, 🟥 = Over budget).
  • An "Edit" mode allows for historical expense modification and quick adjustments to monthly allocations.
  • Data can also be exported to CSV for external analysis.
Technologies Used
HTML CSS Bootstrap 5 JavaScript (Vanilla) LocalStorage
Key Features
  • Offline-first — works without internet after first load
  • Zero cloud dependencies — data stays in users browser
  • Color-coded budgeting feedback
  • Mobile-responsive layout with a lightweight footprint
Use It Yourself

This app is open source and ready for anyone to use — just visit the live site and start tracking your expenses immediately.

  • No sign-up required
  • Fully offline — works right in your browser after the first load
🔒 Security Note:

Your data never leaves your browser. The app uses only LocalStorage of your browser, meaning everything you enter stays entirely on your device. No account, no cloud, no syncing — ensuring complete privacy and control.
